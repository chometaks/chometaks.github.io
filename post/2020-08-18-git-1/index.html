git/github覚え書き②

 Posted on August 18, 2020  |  chometaks

ハッカソンでgit/githubを使ってコード共有したので共有の仕方をメモ

共有の手順

共有する側

  1. 共有したい作業ディレクトリに移動
    $cd hogehoge

  2. gitで初期化
    $git init

  3. github上でリポジトリを作成

  4. リポジトリとデバイスのディレクトリを紐付ける
    $git remote add origin (リポジトリのURL)

  5. github上でコラボレータ依頼を出す


共有してもらう側

  1. github上でコラボレータ登録を受ける

  2. リポジトリをクローン
    $git clone (URL)


編集の手順(共通)

  1. 【最初のみ】ブランチを切る(自分の開発ブランチを作る)
    何か編集を加えるときは必ず！
    $git branch (ブランチ名)

  2. ブランチに入る
    $git checkout (ブランチ名)

  3. 実際の作業

  4. 編集をステージング
    $git add .

  5. 編集をコミット
    $git commit -m "編集内容をわかりやすく"

  6. githubに変更を適用
    $git push origin (ブランチ名)

  7. 【自分の開発が終わったら】masterブランチに自分のブランチの内容を取り込む
    この作業はgithub上で行う

    1. プルリクエストを出す

    2. (リポジトリの管理者)いいと思ったらプルリクエストを受ける

  8. 【他の人がmasterを更新したとき】自分の開発ブランチに最新の内容を取り込む

    1. masterブランチに移動
      $git checkout master

    2. masterブランチを更新
      $git pull origin master

    3. 自分の開発ブランチに移動
      $git checkout (ブランチ名)

    4. masterブランチの内容を取り込む
      $git merge origin master


その他

  • gitに上げたくないファイルは.gitignoreに書く
    →作業ディレクトリに置く、*.jpg など書くことで追跡されない​

  • API keyなど隠したい文字列がある時は別ファイル、環境変数に書く​
    →隠したい文字列を書いたファイルを .gitignore に追記する
    ​ ex)Pythonの場合​
