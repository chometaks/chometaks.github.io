git/github覚え書き①

 Posted on August 9, 2020  |  chometaks

gitとgithubを真面目に使うことから逃げ続けた結果苦労したのでとりあえずメモ(あとで追加するかも)

リポジトリとかブランチとかごっちゃになりそう

ローカルリポジトリ→パソコンの中
リモートリポジトリ→githubサーバ
(リモートリポジトリのデフォルトの名前はorigin)

リモート追跡ブランチ→リモートリポジトリをコピーして追跡する

ローカルリポジトリのブランチ リモート追跡ブランチ リモートリポジトリ
master origin/master master
hoge origin/hoge hoge
huga origin/huga huga

右に行くほど上流ブランチ

git pullがよくわかんない

-pull→fetch+merge
-fetch→リモート追跡ブランチにリモートリポジトリをコピー(すべてのブランチ origin/〇〇 が更新される？)
-merge origin/〇〇→ローカルリポジトリの今いるブランチにリモート追跡ブランチorigin/〇〇をマージ
マージ#とは
両方の変更を保持して合わせること
マージしてもローカルリポジトリが変更されただけなのでリモートリポジトリは変更されてない
参照のみ移動するのはfast forwardマージ

-$git pull
masterブランチにいるとき、リモートのmasterブランチの内容をコピー
masterブランチ以外ではエラー

-$git pull origin master'
上に同じ

-$git pull origin hoge
hogeブランチにいるとき、リモートのhogeブランチの内容をコピー
masterブランチにいるとき、リモートのhogeブランチの内容とマージされる

コミットってなんだ

ローカルリポジトリのブランチの作業内容を編集履歴に入れる
(リモートリポジトリにpushする準備)

ローカルリポジトリの階層

|編集履歴 | |git reset (mode) (commit)↓ ↑git commit -m"コミット内容” | |ステージ | |git reset HEAD ↓ ↑git add . | |作業フォルダ |

実際の流れ(一人でやるとき)

-(始めるとき)ディレクトリをgitで管理できるようにする
.gitファイルが作成される
$git init

-リポジトリを作成

-リポジトリとデバイスのディレクトリを紐付ける $git remote add origin (URL)

-(他のデバイスから持ってくるとき)リモートリポジトリからコピー $git clone (自分のリポジトリのURL)

-(他のデバイスでやった変更を適用) $git pull

-実作業(コマンドラインでファイルを作ったほうがわかりやすいと思う)

-変更をステージング . ですべてのファイル、ファイル名指定もできる
$git add .
(戻すときは'$git reset HEAD’)

-変更をコミット $git commit -m "コミット内容をわかりやすく書く"
(なかったことにするときは$git reset --hard HEAD^)

-リモートリポジトリに変更を適用 $git push -u origin master

ブランチ使うとき

-ブランチ切る $git brunch hoge

-ブランチに入る $git checkout hoge

-(他のデバイスでやったブランチを持ってくるとき) $git pull origin hoge

-実作業

-addしてcommitしてpush

-ブランチの作業内容をmasterにマージ

-masterに入る $git checkout master

-マージ $git merge hoge

-リモートリポジトリにpush $git push -u origin master

次回
